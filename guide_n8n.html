⚙ Configuration pas-à-pas dans n8n
1

Créer un nouveau workflow dans n8n

Dans n8n, cliquez + New Workflow. Nommez-le : "ISQ — Soumission fiches chantier".

2

Nœud 1 — Webhook (déclencheur)

Ajoutez un nœud Webhook. Configurez :

HTTP Method: POST Path: isq-fiches-chantier Authentication: None Response Mode: Immediately Response Code: 200 Response Data: {"status":"ok"}

Copiez l'URL du webhook (format : https://votre-n8n.com/webhook/isq-fiches-chantier). C'est cette URL que vous collez dans index.html → Soumission de fiches au bureau.

3

Coller l'URL dans index.html

Ouvrez index.html sur votre téléphone ou ordinateur. Dans la section 🔗 URLs des Webhooks n8n, collez l'URL dans le champ "📤 Soumission de fiches au bureau". Cliquez Enregistrer. Chaque chef devra faire la même chose sur son téléphone une seule fois.

💡

Le chef peut aussi scanner un QR code de l'URL index.html depuis votre Netlify — il configure ses webhooks une fois et c'est enregistré sur son téléphone.

4

Nœud 2 — Switch (selon type de document)

Ajoutez un nœud Switch après le webhook. Configurez :

Mode: Rules Value to compare: {{ $json.body.document }} // Règle 1 Value: "Bon de travail" → Output 0 // Règle 2 Value: "Fiche de chantier" → Output 1 // Règle 3 Value: "Checklist qualité" → Output 2

Chaque output relie à un nœud Set qui formate l'email spécifique.

5

Nœud 3 — Set (formater le HTML de l'email)

Pour chaque branche, ajoutez un nœud Set qui construit le corps de l'email. Exemple pour Fiche de chantier :

// Nœud Set — Variable "email_html" `<h2 style="color:#0170B9;">📋 Fiche de chantier — {{ $json.body.job_number }}</h2> <p><strong>Date:</strong> {{ $json.body.date_locale }}</p> <p><strong>Client:</strong> {{ $json.body.client_name }}</p> <p><strong>Adresse:</strong> {{ $json.body.client_address }}</p> <hr> <h3>👷 Équipe</h3> <p>Chef: {{ $json.body.chef_nom }}</p> <h3>⏱ Heures</h3> <p>Début: {{ $json.body.heure_debut }} — Fin: {{ $json.body.heure_fin }}</p> <h3>🌤 Météo</h3> <p>{{ $json.body.meteo_conditions }}</p> <h3>📦 Matériaux</h3> <p>{{ $json.body.materiaux_utilises }}</p> <p style="color:#666;font-size:12px;">Soumis le {{ $json.body.date_locale }} à {{ $json.body.heure_locale }}</p>`

Ajustez les noms de champs selon ceux définis dans vos data-jobber-field dans fiche_chantier.html. Tous les champs du formulaire sont dans $json.body.*.

6

Nœud 4 — Gmail / SMTP (envoyer l'email)

Ajoutez un nœud Gmail (ou Email SMTP) après chaque nœud Set. Configurez :

To Email: votre-courriel@innovaspray.ca Subject: 📤 {{ $json.body.document }} — Job #{{ $json.body.job_number }} — {{ $json.body.date_locale }} Email Type: HTML HTML Body: {{ $json.email_html }}
💡

Gmail : Dans n8n Credentials, ajoutez votre compte Gmail (OAuth2). SMTP : Utilisez les paramètres SMTP de votre fournisseur de courriel (ex: Hostinger, cPanel, Microsoft 365).

7

Optionnel — Google Sheets (registre automatique)

Ajoutez un nœud Google Sheets → Append Row après l'email. Crée automatiquement une ligne dans votre tableau de bord :

Spreadsheet: "ISQ — Registre des chantiers" Sheet: "Fiches reçues" Colonnes: A — Date: {{ $json.body.date_locale }} B — Heure: {{ $json.body.heure_locale }} C — Document: {{ $json.body.document }} D — Job #: {{ $json.body.job_number }} E — Client: {{ $json.body.client_name }} F — Chef: {{ $json.body.chef_nom }}
8

Activer le workflow et tester

Dans n8n, cliquez Activate (toggle en haut à droite). Puis sur votre cellulaire, ouvrez fiche_chantier.html, remplissez quelques champs, et cliquez 📤 Soumettre au bureau. Vous devriez recevoir l'email dans les secondes qui suivent.

⚠

En mode Test dans n8n, l'URL du webhook change (elle contient /webhook-test/). Pour la production, utilisez toujours l'URL sans -test, qui n'est active que quand le workflow est activé.

Exemple de payload JSON reçu par n8n (Fiche de chantier)

🔑 Configuration de la clé API Jobber
⚠

Sécurité : Ne jamais partager votre clé API Jobber. Ne jamais la copier dans le code source. Utilisez les variables d'environnement n8n (Settings → Credentials) pour stocker la clé en toute sécurité.

1

Accéder aux paramètres développeur Jobber

Dans Jobber, allez à Gear icon (⚙) → Settings → Developer Tools → API Keys. Si l'option n'est pas visible, activez-la dans Settings → App Marketplace → Jobber API.

2

Créer une nouvelle clé API

Cliquez sur New API Key. Donnez-lui un nom descriptif (ex: "n8n Automatisation"). Sélectionnez les permissions requises : Quotes (read), Jobs (read/write), Clients (read), Invoices (read), Webhooks (read/write).

3

Copier la clé et l'URL de base

Notez votre clé API (format Bearer Token). L'URL de base de l'API Jobber est :
https://api.getjobber.com/api/graphql (GraphQL) ou
https://api.getjobber.com/api/v1 (REST selon votre plan)

4

Configurer le credential dans n8n

Dans n8n: Credentials → New → HTTP Header Auth. Nom: "Jobber API". Header Name: Authorization. Header Value: Bearer VOTRE_CLE_API_ICI. Enregistrer et tester la connexion.

5

Configurer les webhooks Jobber

Dans Jobber Developer Tools → Webhooks → New Webhook. Pour chaque événement voulu, entrez l'URL de votre workflow n8n. Format d'URL n8n: https://votre-n8n.com/webhook/nom-du-workflow. Sélectionnez les événements (quote.approved, job.completed, invoice.overdue).

6

Tester avec un événement réel ou simulé

Jobber permet d'envoyer un payload de test depuis Developer Tools → Webhooks → Send Test. Dans n8n, activez le workflow en mode "Test" pour voir les données reçues avant de passer en production.

Exemple de requête HTTP Request n8n — GET Job Details
// Nœud HTTP Request — Récupérer détails d'un job Method: GET URL: https://api.getjobber.com/api/v1/jobs/{{ $json.job_id }} Headers: Authorization: Bearer {{ $credentials.jobberApi.key }} Content-Type: application/json X-JOBBER-GRAPHQL-VERSION: 2024-01-15 // Réponse attendue (résumé) { "id": 12345, "job_number": "JOB-2024-001", "status": "completed", "client": { "name": "Jean Dupont", "email": "jean@email.com" }, "total": 3450.00, "completed_at": "2024-10-15T16:30:00Z" }
