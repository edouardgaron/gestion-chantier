📋

Comment ça fonctionne : L'employé entre un numéro de projet dans n'importe quel document HTML (Fiche de chantier, Soumission, etc.). Le document envoie ce numéro au workflow n8n, qui cherche la ligne correspondante dans le Google Sheets « Projets », puis retourne toutes les informations directement dans les champs du document — automatiquement, en quelques secondes.

🔩 Explication de chaque nœud
1
🔗

Webhook — Lookup projet

n8n-nodes-base.webhook — Méthode : GET
🎯 Rôle

C'est la porte d'entrée du workflow. Il crée une URL publique que les documents HTML peuvent appeler. Dès qu'une requête arrive, il démarre l'exécution.

⚙️ Paramètres clés
  • Méthode : GET
  • Chemin : sheets-lookup
  • Mode réponse : responseNode (répond plus tard)
  • CORS : autorisé depuis tous les domaines (*)
📤 Sortie
  • query.numero — le n° entré par l'employé
  • Ex: ?numero=1042
  • URL complète : /webhook/sheets-lookup?numero=1042
💡

L'URL du webhook change selon que le workflow est en test (/webhook-test/sheets-lookup) ou en production (/webhook/sheets-lookup). Les documents HTML doivent pointer vers l'URL de production une fois le workflow activé.

2
🧹

Nettoyer le numéro de projet

n8n-nodes-base.code — JavaScript
🎯 Rôle

Normalise le numéro entré par l'employé pour éviter les erreurs. Peu importe le format saisi, on extrait les chiffres et on standardise la valeur pour la recherche.

⚙️ Formats acceptés
  • 1042 → numéro brut
  • #PRJ-1042 → avec préfixe
  • PRJ 1042 → avec espace
  • 1042 → avec espaces
📤 Sortie produite
  • numero_brut — valeur originale
  • numero — chiffres seulement
  • numero_norm — majuscules, sans espaces
💻 Ce que fait le code
// Entrée: " #PRJ-1042 " const raw = ($json.query?.numero || '').toString().trim(); // → "1042" (chiffres extraits pour comparaison) const numeroChiffres = raw.replace(/[^0-9]/g, ''); // → "PRJ-1042" (normalisé pour comparaison exacte) const numeroNorm = raw.toUpperCase().replace(/\s+/g, ''); // Si vide → erreur immédiate avec message clair if (!raw) throw new Error('Numéro de projet manquant');
3
📊

Google Sheets — Lire les projets

n8n-nodes-base.googleSheets v4.5 — Opération : Read
🎯 Rôle

Se connecte à votre Google Sheets « Projets » et lit toutes les lignes. Chaque ligne devient un item séparé dans n8n. La première ligne du Sheets doit contenir les en-têtes de colonnes.

⚙️ Paramètres à configurer
  • Credential : Google Sheets OAuth2 (à créer dans n8n)
  • Document ID : ID du Google Sheets (dans l'URL)
  • Sheet Name : Nom de l'onglet (ex: Projets)
  • Value Render : UNFORMATTED_VALUE
📤 Sortie

Chaque ligne du Sheets = 1 item. Les colonnes deviennent des propriétés JSON portant le nom de l'en-tête :

  • Numéro de projet
  • Nom du client
  • Sous-total, etc.
🔑

Comment trouver l'ID du Sheets : Ouvrez votre Google Sheets. Dans l'URL, copiez la partie entre /d/ et /edit. Ex: https://docs.google.com/spreadsheets/d/1BxiMVs0XRA5nFMdKvBdBZjgmUUqptlbs74OgVE2upms/edit

✅

Connexion Google : Dans n8n → Settings → Credentials → New → Google Sheets OAuth2. Suivez le guide d'autorisation Google. Une fois créé, sélectionnez-le dans ce nœud.

4
🔎

Trouver la ligne du projet

n8n-nodes-base.code — Mode : Run Once For All Items
🎯 Rôle

Parcourt toutes les lignes retournées par Google Sheets et trouve celle dont le numéro de projet correspond exactement à ce que l'employé a entré. Compare de deux façons pour être flexible.

⚙️ Logique de recherche
  • Compare les chiffres seulement (robuste)
  • OU compare le texte normalisé (exact)
  • Si aucune correspondance → erreur 404 avec message
  • Mode runOnceForAllItems = traite tout d'un coup
📤 Sortie

Retourne 1 seul item : la ligne du projet trouvé avec toutes ses colonnes.

  • Si trouvé → données du projet
  • Si non trouvé → erreur capturée par le nœud 7
💻 Ce que fait le code
// Récupérer le numéro nettoyé depuis le nœud 2 const numero = $('Nettoyer le numéro de projet').first().json.numero; // Comparer chaque ligne du Sheets const match = allRows.find(item => { const val = item.json['Numéro de projet']; return valChiffres === numero // ex: "1042" === "1042" || valNorm === numeroNorm; // ex: "PRJ-1042" === "PRJ-1042" }); if (!match) throw new Error('Projet introuvable dans le Sheets');
5
⚙️

Transformer → Champs documents

n8n-nodes-base.code — Mapping colonnes → champs HTML
🎯 Rôle

Convertit les noms de colonnes du Sheets vers les noms de champs standardisés attendus par les documents HTML. Calcule aussi les taxes et formatte les montants en dollars CAD.

⚙️ Calculs automatiques
  • TPS (5%): sous-total × 0,05
  • TVQ (9,975%): sous-total × 0,09975
  • Total TTC: sous-total + TPS + TVQ
  • Solde dû: Total − Acompte
  • Formatage dates en AAAA-MM-JJ
📤 Champs produits
  • job_number, client_name, client_company
  • client_phone, client_email, job_address
  • start_date, work_type, notes
  • foreman_name, duration_days, crew_size
  • subtotal, tps_amount, tvq_amount
  • total_ttc, acompte_amount, solde_amount
🔗

Ces noms de champs (job_number, client_name, etc.) correspondent exactement aux attributs data-jobber-field dans les documents HTML. C'est ce qui permet le remplissage automatique des formulaires.

6
✅

Répondre — JSON vers le document

n8n-nodes-base.respondToWebhook — Code 200
🎯 Rôle

Envoie la réponse JSON au document HTML qui avait fait la requête. Le document reçoit les données et remplit automatiquement tous les champs correspondants.

⚙️ Paramètres
  • respondWith : json
  • responseBody : ={{ $json }}
  • CORS : Access-Control-Allow-Origin: *
  • Code HTTP : 200 (succès)
📤 Format de réponse
{ "job_number": "1042", "client_name": "Jean Tremblay", "job_address": "123 rue St-Paul", "total_ttc": "2 345,67 $", // ... tous les autres champs }
7
❌

Répondre — Erreur projet introuvable

n8n-nodes-base.respondToWebhook — Code 404
🎯 Rôle

Se déclenche si le numéro de projet n'existe pas dans le Google Sheets, ou si le numéro entré est invalide. Retourne un message d'erreur clair au document HTML.

⚙️ Paramètres
  • Code HTTP : 404
  • Message : « Projet introuvable dans le Google Sheets »
  • Le document HTML affichera ce message à l'employé
📤 Réponse retournée
{ "erreur": "Projet introuvable dans le Google Sheets. Vérifiez le numéro." }
⚠️

À configurer dans n8n : Dans les connexions du workflow, reliez le chemin d'erreur (branche rouge) du nœud 4 vers ce nœud 7. Ainsi, quand un projet est introuvable, l'erreur est capturée et retournée proprement.

🚀 Étapes de mise en place
1

Préparer le Google Sheets

Créez ou ouvrez votre Google Sheets. Renommez l'onglet en Projets. Ajoutez les en-têtes de colonnes exactement comme dans le tableau ci-dessus. Entrez vos projets, une ligne par projet.

2

Copier l'ID du Sheets

Dans l'URL du Sheets, copiez la valeur entre /d/ et /edit. Exemple : 1BxiMVs0XRA5nFMdKvBdBZjgmUUqptlbs74OgVE2upms. Vous en aurez besoin à l'étape 4.

3

Créer le credential Google Sheets dans n8n

Dans n8n → Settings → Credentials → New → Google Sheets OAuth2. Suivez l'assistant d'autorisation. Une fois créé, notez son nom exact.

4

Importer le workflow et le configurer

Dans n8n → Workflows → Import from JSON → collez le contenu de workflow_sheets_lookup.json. Ouvrez le nœud Google Sheets — Lire les projets et remplacez VOTRE_SHEET_ID_ICI par votre vrai ID. Sélectionnez votre credential Google.

5

Connecter le nœud d'erreur

Dans le canvas n8n, reliez le chemin d'erreur (point rouge) du nœud Trouver la ligne du projet vers le nœud Répondre — Erreur. Faites la même chose pour le nœud Nettoyer le numéro.

6

Tester avec un numéro réel

Activez d'abord le workflow en mode test. Ouvrez un document HTML (ex: Fiche de Chantier), entrez un numéro de projet existant dans votre Sheets, et cliquez sur le bouton de recherche. Les champs devraient se remplir automatiquement.

7

Mettre en production

Dans n8n, activez le workflow (toggle en haut à droite). L'URL de production devient : https://VOTRE-N8N.com/webhook/sheets-lookup. Mettez à jour cette URL dans le code JavaScript de chaque document HTML.

🔗

Source unique de données : Ce workflow Google Sheets est la source principale de tous vos projets. Les documents HTML lisent les données directement depuis votre tableur PRODUCTION via ce workflow n8n.
